**scripts/11-verify-gmail-auth.sh:** Create a verification script that:

1. Display header: "ClawBot -- Gmail Auth Verification"
2. Checks gog auth has both accounts:
   ```bash
   ACCOUNTS=$(gog auth list --json)
   ACCOUNT_COUNT=$(echo "$ACCOUNTS" | jq 'length')
   if [ "$ACCOUNT_COUNT" -lt 2 ]; then
     echo "FAIL: Expected 2 accounts, found $ACCOUNT_COUNT"
     exit 1
   fi
   ```
3. For each account, verify Gmail API access by fetching the 3 most recent inbox messages:
   ```bash
   echo "Testing personal account API access..."
   PERSONAL_EMAIL=$(jq -r '.email' /sandbox/state/email-sync-personal.json)
   gog gmail messages search "in:inbox" --max 3 --account "$PERSONAL_EMAIL"

   echo "Testing work account API access..."
   WORK_EMAIL=$(jq -r '.email' /sandbox/state/email-sync-work.json)
   gog gmail messages search "in:inbox" --max 3 --account "$WORK_EMAIL"
   ```
4. Verify history.list works for each account:
   ```bash
   PERSONAL_HID=$(jq -r '.historyId' /sandbox/state/email-sync-personal.json)
   echo "Testing personal account history.list with historyId=$PERSONAL_HID..."
   gog gmail history --account "$PERSONAL_EMAIL" --since "$PERSONAL_HID"

   WORK_HID=$(jq -r '.historyId' /sandbox/state/email-sync-work.json)
   echo "Testing work account history.list with historyId=$WORK_HID..."
   gog gmail history --account "$WORK_EMAIL" --since "$WORK_HID"
   ```
5. Verify state files exist and have valid content:
   ```bash
   for ACCOUNT in personal work; do
     FILE="/sandbox/state/email-sync-${ACCOUNT}.json"
     if [ ! -f "$FILE" ]; then
       echo "FAIL: $FILE missing"
       exit 1
     fi
     HID=$(jq -r '.historyId' "$FILE")
     if [ "$HID" = "null" ] || [ -z "$HID" ]; then
       echo "FAIL: $FILE has null historyId"
       exit 1
     fi
     echo "OK: $FILE has historyId=$HID"
   done
   ```
6. Verify memory directories exist:
   ```bash
   for ACCOUNT in personal work; do
     DIR="/sandbox/memory/email-patterns/${ACCOUNT}"
     if [ ! -d "$DIR" ]; then
       echo "FAIL: $DIR missing"
       exit 1
     fi
     echo "OK: $DIR exists"
   done
   ```
7. Display a summary of all checks (pass/fail counts) and exit 0 on all pass, exit 1 on any failure.

**Step 1: GCP Console setup (one-time, ~10 minutes)**
1. Go to https://console.cloud.google.com/
2. Create a new project (or select existing) named "ClawBot"
3. Enable Gmail API: APIs and Services -> Library -> search "Gmail API" -> Enable
4. Enable Google Calendar API: same process
5. Configure OAuth consent screen:
   - User type: External
   - Fill in app name ("ClawBot"), user support email
   - Add scopes: gmail.readonly, gmail.compose, gmail.modify, calendar.readonly
   - CRITICAL: Set publishing status to **Production** (NOT Testing)
   - If it was in Testing mode, you MUST create NEW credentials after switching (Pitfall #7)
6. Create OAuth 2.0 Client ID:
   - Application type: Desktop app
   - Name: "ClawBot Desktop"
   - Download the client_secret.json file
7. Copy GMAIL_OAUTH_CLIENT_ID and GMAIL_OAUTH_CLIENT_SECRET to your `.env` file
8. Copy client_secret.json to `/sandbox/config/client_secret.json`

**Step 2: Run setup script (in WSL2)**
```bash
bash scripts/10-gmail-oauth-setup.sh
```
Follow the prompts to authenticate both Gmail accounts in the browser.

**Step 3: Run verification script**

