**Context packaging (D-08: metadata + classification only):**
- `buildDelegationContext(classifiedEmail)` -- returns object with fields: `sender`, `subject`, `snippet`, `account`, `threadId`, `messageId`, `classification: { categories: classifiedEmail.categories, confidence: classifiedEmail.categories[0].confidence, reasoning: classifiedEmail.reasoning, recommended_action: classifiedEmail.recommended_action }`. Does NOT include full email body -- target agent fetches via Gmail API if needed.

**Task description builder:**
- `buildTaskDescription(classifiedEmail, agentId)` -- returns a string prompt for the target agent. Format:
```
Email triage delegation from comms agent.

Sender: {sender}
Subject: {subject}
Account: {account}
Category: {primary category} (confidence: {confidence})
Recommended action: {recommended_action}

Classification reasoning: {reasoning}

Snippet: {snippet}

Please review this email and take appropriate action. If you need the full email body, fetch it via Gmail API using messageId: {messageId} (threadId: {threadId}).
```

**Delegation execution:**
- `delegateToAgent(classifiedEmail, sessionSpawnFn)` -- the main entry point:
  1. Get primary category from `classifiedEmail.categories[0].category`.
  2. Look up `agentId` from `DELEGATION_ROUTING[category]`. If not found, return `{ skipped: true, reason: 'no_delegation_target' }`.
  3. Build task description with `buildTaskDescription(classifiedEmail, agentId)`.
  4. Call `sessionSpawnFn({ agentId, task, label: 'email-' + classifiedEmail.messageId, runTimeoutSeconds: DELEGATION_FOLLOW_UP_HOURS * 3600 })`.
     - `sessionSpawnFn` is passed as parameter (same callback pattern as `telegramSendFn`) for testability. In production, this wraps OpenClaw's `sessions_spawn` tool.
  5. If spawn succeeds (returns `{ status: 'accepted', runId, childSessionKey }`):
     - Add entry to delegation queue: `{ id: generateId(), messageId, threadId, account, target_agent: agentId, context: buildDelegationContext(classifiedEmail), delegated_at: new Date().toISOString(), runId, childSessionKey, retry_count: 0, max_retries: DELEGATION_MAX_RETRIES, follow_up_at: (now + DELEGATION_FOLLOW_UP_HOURS hours).toISOString(), status: 'active' }`.
     - Save queue state.
     - Return `{ delegated: true, agentId, runId }`.
  6. If spawn fails (throws error or returns non-accepted status):
     - Add to queue with `status: 'queued'`, `retry_count: 0`, `next_retry_at: (now + DELEGATION_RETRY_DELAY_MINUTES minutes).toISOString()`.
     - Save queue state.
     - Return `{ queued: true, agentId, reason: err.message || 'spawn_failed' }`.
  Wrap in try/catch. On unexpected error, log and return `{ error: true, message: err.message }`.

**Queue processing (D-09: retry logic):**
- `processDelegationQueue(sessionSpawnFn, telegramSendFn)` -- called every heartbeat cycle:
  1. Load queue state.
  2. For each item in `pending` with `status === 'queued'` and `next_retry_at <= now`:
     a. Attempt `sessionSpawnFn({ agentId: item.target_agent, task: buildTaskDescription(item.context, item.target_agent), label: 'email-' + item.messageId, runTimeoutSeconds: DELEGATION_FOLLOW_UP_HOURS * 3600 })`.
     b. On success: update status to `'active'`, set `runId`, `childSessionKey`, `follow_up_at`, increment `retry_count`.
     c. On failure: increment `retry_count`. If `retry_count >= max_retries`, move to `dead_letter` array, set `status: 'dead_letter'`, and send Telegram notification: `"Delegation failed: [{subject}] to @{target_agent} after {retry_count} attempts. Taking no further action -- please handle manually."` via `telegramSendFn`.
     d. If retries remain: set `next_retry_at` to `now + DELEGATION_RETRY_DELAY_MINUTES minutes`.
  3. Save queue state.
  4. Return `{ retried: count, dead_lettered: count }`.

**Follow-up tracking (D-10: 2-hour timeout):**
- `checkFollowUps(telegramSendFn)` -- called every heartbeat cycle:
  1. Load queue state.
  2. For each item in `pending` with `status === 'active'` and `follow_up_at <= now`:
     - Send Telegram nudge: `"@{target_agent} hasn't responded to \"{subject}\" (delegated {hours_ago}h ago). Nudge the agent or take over?"` via `telegramSendFn(message, 'HTML')`.
     - Update `follow_up_at` to `now + DELEGATION_FOLLOW_UP_HOURS hours` (prevent re-nudging for another 2h).
  3. Save queue state.
  4. Return `{ nudged: count }`.

**Delegation completion handler:**
- `markDelegationComplete(runId, result)` -- called when OpenClaw announces sub-agent completion:
  1. Load queue state.
  2. Find entry in `pending` with matching `runId`.
  3. Set `status: 'completed'`, `completed_at: now`, `result: result`.
  4. Save queue state.
  5. Return the entry (for Telegram result notification in delivery layer).
