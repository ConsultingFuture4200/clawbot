**2. Create sandbox/skills/classify-email/draft-generator.js** with these functions:

**OAuth/Gmail client (reuse Phase 1 pattern from gmail-oauth-helper.cjs):**
- `getOAuth2Client(account)` -- creates OAuth2Client from `/sandbox/config/client_secret.json` and loads tokens from `/sandbox/state/token-{account}.json`. Uses `const { OAuth2Client } = require('google-auth-library');`. Returns configured OAuth2Client with credentials set.
- `getGmailClient(account)` -- creates `new gmail_v1.Gmail({ auth: getOAuth2Client(account) })` using `const { gmail_v1 } = require('@googleapis/gmail');`. Returns gmail client.

**MIME construction (per Research Pattern 2 -- Pitfall 3):**
- `buildReplyMime({ to, from, subject, body, inReplyTo, references })` -- builds RFC 2822 MIME message. Headers: From, To, Subject (prepend "Re: " if not already present), MIME-Version: 1.0, Content-Type: text/plain; charset=utf-8. If `inReplyTo` provided, add `In-Reply-To` and `References` headers. Join headers with `\r\n`, append `\r\n\r\n` + body. Return `Buffer.from(message).toString('base64url')`.

**Message-ID retrieval (per Research Open Question 3):**
- `fetchOriginalMessageId(account, messageId)` -- calls `gmail.users.messages.get({ userId: 'me', id: messageId, format: 'metadata', metadataHeaders: ['Message-ID', 'From'] })`. Extracts and returns `{ messageIdHeader, fromHeader }` from response headers array. Costs 5 quota units.

**Calendar conflict detection (per Research Pattern -- Calendar):**
- `checkCalendarConflict(account, eventStart, eventEnd)` -- uses `const { google } = require('googleapis');` to create `google.calendar({ version: 'v3', auth: getOAuth2Client(account) })`. Calls `calendar.freebusy.query({ requestBody: { timeMin: eventStart, timeMax: eventEnd, timeZone: 'America/Los_Angeles', items: [{ id: 'primary' }] } })`. Returns `{ isFree: boolean, conflicts: Array<{ start, end }> }`. If eventStart/eventEnd not parseable from email, returns `{ isFree: null, conflicts: [] }` (unknown).

**Draft creation:**
- `createGmailDraft(account, { threadId, raw })` -- calls `gmail.users.drafts.create({ userId: 'me', requestBody: { message: { raw, threadId } } })`. Returns `{ draftId: response.data.id, messageId: response.data.message.id, threadId: response.data.message.threadId }`.
- `updateGmailDraft(account, draftId, { threadId, raw })` -- calls `gmail.users.drafts.update({ userId: 'me', id: draftId, requestBody: { message: { raw, threadId } } })`. Returns same shape.
- `deleteGmailDraft(account, draftId)` -- calls `gmail.users.drafts.delete({ userId: 'me', id: draftId })`. Wraps in try/catch; logs but does not throw on 404 (draft already deleted).

**Draft tracker (per Research Pattern -- Draft Tracker):**
- `hasDraftForThread(threadId)` -- loads `DRAFT_TRACKER_PATH`, checks if `drafts[threadId]` exists with `status === 'pending'`. Returns boolean.
- `getDraftForThread(threadId)` -- loads `DRAFT_TRACKER_PATH`, returns `drafts[threadId]` if it exists with `status === 'pending'`, otherwise `null`. Used by `generateDraft` to retrieve existing draft info for update path.
- `trackDraft(threadId, draftInfo)` -- loads tracker, sets `drafts[threadId]` to `{ draftId, account, category, subject, short_key: draftId.slice(0, 12), created_at: new Date().toISOString(), expires_at: (now + 48h).toISOString(), status: 'pending', snooze_until: null, telegram_msg_id: null }`. Saves state. **CRITICAL:** The `short_key` field is required -- Plan 03 Task 1 `buildDraftApprovalKeyboard` uses it, and Plan 03 Task 2 `handleCallbackQuery` looks up drafts by `short_key`.
- `updateDraftStatus(threadId, status)` -- loads tracker, sets `drafts[threadId].status = status`. Saves state.
- `getExpiredDrafts()` -- loads tracker, filters entries where `status === 'pending'` and `expires_at < now` and (`snooze_until === null` or `snooze_until < now`). Returns array of `{ threadId, draftId, account, subject }`.
- `cleanupExpiredDrafts()` -- calls `getExpiredDrafts()`, for each calls `deleteGmailDraft(account, draftId)`, then sets status to `'expired'`. Returns array of cleaned-up subjects for notification.

**Main orchestrator:**
- `generateDraft(classifiedEmail)` -- the primary entry point for auto-drafting (D-03, D-16, D-18):
  1. Check `hasDraftForThread(classifiedEmail.threadId)` -- if true, UPDATE the existing draft rather than skipping (per D-18: "update existing draft rather than creating duplicate"):
     a. Call `getDraftForThread(classifiedEmail.threadId)` to get existing `{ draftId, account }`.
     b. Generate new draft text via `getDraftText(classifiedEmail, conflictResult)` (same steps 2-5 below apply for text generation).
     c. Call `fetchOriginalMessageId(account, classifiedEmail.messageId)` for threading headers.
     d. Build new MIME with `buildReplyMime(...)`.
     e. Call `updateGmailDraft(account, existingEntry.draftId, { threadId, raw })` to replace the draft content.
     f. Call `updateDraftStatus(classifiedEmail.threadId, 'pending')` to reset status (in case it was snoozed/editing).
     g. Update `short_key` in tracker if draftId changed: load tracker, set `drafts[threadId].draftId = updatedResult.draftId`, `drafts[threadId].short_key = updatedResult.draftId.slice(0, 12)`, save.
     h. Return `{ updated: true, draftId: updatedResult.draftId, threadId, account, category, draftText }`.
  2. Determine primary category from `classifiedEmail.categories[0].category`.
  3. If category not in `DRAFT_CATEGORIES`, return `{ skipped: true, reason: 'no_draft_category' }`.
  4. If category is `'calendar'`: extract event times from snippet (basic ISO date regex), call `checkCalendarConflict(account, start, end)`. Pass result to `getDraftText`.
  5. Call `getDraftText(classifiedEmail, conflictResult)` from draft-templates.js. If returns null, return skipped.
  6. Call `fetchOriginalMessageId(account, classifiedEmail.messageId)` to get In-Reply-To header.
  7. Build MIME with `buildReplyMime({ to: originalFrom, from: accountEmail, subject: classifiedEmail.subject, body: draftText, inReplyTo: originalMsgId })`.
  8. Call `createGmailDraft(account, { threadId, raw })`.
  9. Call `trackDraft(threadId, { draftId, account, category, subject })`.
  10. Return `{ created: true, draftId, threadId, account, category, draftText }`.
  Wrap entire function in try/catch. On error, log `[draft-generator] Error creating draft: ${err.message}` and return `{ error: true, message: err.message }`.

- `generateSmartDraft(classifiedEmail)` -- wrapper for urgent emails (D-04). Same flow as `generateDraft` but forces `generateSmartDraftText` regardless of category. Used only for `urgent` emails.

Export all public functions (including `getDraftForThread`).
