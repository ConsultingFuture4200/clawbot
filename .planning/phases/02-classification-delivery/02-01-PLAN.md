  <action>
    **File 1: sandbox/config/classification-schema.json**

    Create the Gemini responseSchema JSON file. This is passed directly to `config.responseSchema` in the @google/genai SDK call. Structure per 02-RESEARCH.md Pattern 2:

    ```json
    {
      "type": "object",
      "properties": {
        "classifications": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "email_index": { "type": "integer" },
              "categories": {
                "type": "array",
                "items": {
                  "type": "object",
                  "properties": {
                    "category": {
                      "type": "string",
                      "enum": ["code", "calendar", "research", "home", "urgent", "routine", "spam_noise"]
                    },
                    "confidence": { "type": "number" }
                  },
                  "required": ["category", "confidence"]
                }
              },
              "reasoning": { "type": "string" },
              "recommended_action": { "type": "string" },
              "is_urgent": { "type": "boolean" }
            },
            "required": ["email_index", "categories", "reasoning", "recommended_action", "is_urgent"]
          }
        }
      },
      "required": ["classifications"]
    }
    ```

    Key points:
    - 7 categories enum: code, calendar, research, home, urgent, routine, spam_noise (underscore, not slash — JSON schema enum values)
    - categories is an array for multi-label support (CLASS-03)
    - reasoning field for chain-of-thought (CLASS-02, D-07)
    - email_index integer for batch correlation (Pitfall 6 mitigation)
    - is_urgent boolean for immediate notification routing (D-02)

    **File 2: sandbox/config/classification-examples.json**

    Create 21 few-shot examples (3 per category) per D-06. Each example has: sender, subject, snippet, expected_categories (array of {category, confidence}), reasoning, recommended_action, is_urgent. Examples must be realistic and diverse:

    **code** (3): GitHub PR review request, CI build failure notification, npm security advisory
    **calendar** (3): Meeting invitation, meeting reschedule, RSVP reminder
    **research** (3): ArXiv paper alert, newsletter from tech publication, research digest
    **home** (3): Smart home alert, utility bill notification, package delivery tracking
    **urgent** (3): Server down alert (urgent+code), urgent meeting change (urgent+calendar), security breach notification (urgent only)
    **routine** (3): Order confirmation, subscription renewal, account statement
    **spam_noise** (3): Marketing promotion, cold outreach, survey request

    Note: The 3 "urgent" examples should demonstrate multi-label (urgent co-occurring with code, calendar, or standalone) per CLASS-03.

    **File 3: sandbox/skills/classify-email/types.js**

    Create the shared constants and prompt builder module (CommonJS):

    ```javascript
    // Constants
    const CATEGORIES = ['code', 'calendar', 'research', 'home', 'urgent', 'routine', 'spam_noise'];

    const CONFIDENCE_THRESHOLDS = {
      AUTO_LABEL: 0.85,      // >0.85: auto-label, shown in digest, no prompt (D-01)
      ACT_AND_CONFIRM: 0.70, // 0.70-0.84: act-and-confirm, silent confirm if ignored (D-01, D-03)
      ASK_USER: 0.70,        // <0.70: ask user, shown with '?' marker (D-01)
    };

    const BATCH_SIZE = 5; // D-05: 5 emails per Gemini request

    const GEMINI_MODEL = 'gemini-3-flash';
    const OLLAMA_MODEL = 'qwen2.5:7b';
    const OLLAMA_URL = 'http://127.0.0.1:11434';
    ```

    Functions to export:
    - `loadClassificationSchema()` — reads and returns sandbox/config/classification-schema.json
    - `loadFewShotExamples()` — reads and returns sandbox/config/classification-examples.json
    - `buildClassificationPrompt(emails, fewShotExamples)` — returns { systemPrompt, userPrompt } for Gemini. System prompt includes rules (multi-label, confidence 0-1, urgent can co-occur, CoT required) and few-shot examples. User prompt lists emails by index with sender/subject/snippet. Per RESEARCH.md classification prompt template.
    - `buildSpamGatePrompt(sender, subject, snippet)` — returns the messages array for Ollama /api/chat. System message: "You are a spam detector. Classify the email as spam or not_spam based on sender and subject." User message: formatted sender/subject/snippet. Per RESEARCH.md Pattern 3.
    - `getConfidenceTier(confidence)` — returns 'auto_label' | 'act_and_confirm' | 'ask_user' based on thresholds (D-01)

    All file paths should use `/sandbox/config/` prefix (the sandbox runtime path).
    Module.exports all constants and functions.
