Output: spam-gate.js (Ollama binary filter), classifier.js (Gemini batch classifier + sender cache + confidence routing), initialized sender cache files.
</objective>

  <action>
    Create `sandbox/skills/classify-email/spam-gate.js` (CommonJS module).

    **Function: `runSpamGate(email)`**
    - Input: `{ sender: string, subject: string, snippet: string }`
    - Calls Ollama `/api/chat` at `OLLAMA_URL` (from types.js) with:
      - model: `OLLAMA_MODEL` (from types.js, = 'qwen2.5:7b')
      - messages: from `buildSpamGatePrompt(email.sender, email.subject, email.snippet)` (from types.js)
      - stream: false
      - format: JSON schema `{ type: "object", properties: { is_spam: { type: "boolean" }, reason: { type: "string" } }, required: ["is_spam"] }`
    - Parse response: `JSON.parse(result.message.content)` to get `{ is_spam, reason }`
    - Return: `{ is_spam: boolean, reason: string }`
    - Error handling:
      - If Ollama is not responding (ECONNREFUSED, timeout after 30s), return `{ is_spam: false, reason: 'ollama_unavailable' }` — per D-04, if spam gate fails, email proceeds to Gemini (fail open, not fail closed)
      - If JSON parse fails, return `{ is_spam: false, reason: 'parse_error' }`
      - Log all errors to console.error with `[spam-gate]` prefix

    **Function: `runSpamGateBatch(emails)`**
    - Input: array of `{ sender, subject, snippet }` objects
    - Runs `runSpamGate` sequentially for each email (Ollama handles one request at a time efficiently on local GPU)
    - Returns: array of `{ email, spamResult }` where spamResult has `{ is_spam, reason }`
    - Splits result into two arrays: `{ spam: [...], notSpam: [...] }`

    Export: `module.exports = { runSpamGate, runSpamGateBatch };`

    Note: Do NOT auto-archive spam. Per RESEARCH.md anti-patterns, Phase 2 only classifies and flags. Auto-archive is Phase 4 (SPAM-03). Spam emails are logged and skipped from Gemini classification, but not deleted or archived.
  </action>

    **File 3: sandbox/skills/classify-email/classifier.js** (CommonJS module)

    Imports:
    - `const { GoogleGenAI } = require('@google/genai');`
    - `const fs = require('fs');`
    - `const path = require('path');`
    - `const { BATCH_SIZE, GEMINI_MODEL, CONFIDENCE_THRESHOLDS, loadClassificationSchema, loadFewShotExamples, buildClassificationPrompt, getConfidenceTier } = require('./types');`

    Constants:
    - `const STATE_DIR = '/sandbox/state';`
    - `const GEMINI_DELAY_MS = 6500;` (6.5s between requests to stay under 10 RPM per Pitfall 1)
    - `const SENDER_CACHE_MAX = 10000;` (cap per Pitfall 4)

    **Function: `classifyBatch(emails)`**
    - Input: array of up to BATCH_SIZE (5) email objects `{ sender, subject, snippet }`
    - Creates `GoogleGenAI` instance with `process.env.GEMINI_API_KEY`
    - Loads schema via `loadClassificationSchema()`
    - Loads examples via `loadFewShotExamples()`
    - Builds prompt via `buildClassificationPrompt(emails, examples)`
    - Calls `ai.models.generateContent({ model: GEMINI_MODEL, contents: userPrompt, config: { systemInstruction: systemPrompt, responseMimeType: 'application/json', responseSchema: schema } })`
    - Parses `JSON.parse(response.text)` to get classifications
    - **Validation** (Pitfall 6 mitigation): Check `result.classifications.length === emails.length`. If mismatch, identify missing email_index values. For each missing index, make an individual classification request as fallback (log warning).
    - For each classification, compute `confidenceTier` using `getConfidenceTier(classification.categories[0].confidence)` and attach to result
    - Return: array of enriched classification objects with added `confidenceTier` field
    - Error handling:
      - 429 (rate limit): log warning `[classifier] Gemini rate limited, queuing for next cycle`, return null (caller must handle)
      - Other API errors: log error, return null
      - JSON parse error: log error, return null

    **Function: `classifyPipeline(emails, account)`**
    - Input: array of ANY size of email objects, account string ('personal' | 'work')
    - Chunks emails into groups of BATCH_SIZE (5)
    - For each chunk, calls `classifyBatch(chunk)` with `GEMINI_DELAY_MS` delay between chunks
    - After classification, for each email:
      - Call `isUnknownSender(email.sender, account)` to check sender cache
      - Call `updateSenderCache(email.sender, account)` to record sender
      - Attach `is_unknown_sender: boolean` to result
      - Attach `account: string` to result
      - Per D-02: if `classification.is_urgent === true`, set `delivery: 'immediate'` regardless of confidence
      - Per D-01: if not urgent, set `delivery: 'batch'`
    - Return: `{ classified: [...enrichedResults], failed: [...emailsThatFailedClassification] }`

    **Function: `isUnknownSender(senderEmail, account)`**
    - Loads sender cache from `${STATE_DIR}/sender-cache-${account}.json`
    - Returns `true` if senderEmail not in `cache.senders` (D-08)

    **Function: `updateSenderCache(senderEmail, account)`**
    - Loads sender cache from `${STATE_DIR}/sender-cache-${account}.json`
    - If sender exists: increment `count`
    - If sender does not exist: add `{ first_seen: new Date().toISOString(), count: 1 }`
    - If cache size exceeds SENDER_CACHE_MAX (10000): evict oldest entries by `first_seen` date until under limit (Pitfall 4)
    - Update `last_updated` timestamp
    - Write cache back to disk

    **Function: `loadSenderCache(account)`** (internal helper)
    - Reads `${STATE_DIR}/sender-cache-${account}.json`
    - Returns parsed JSON, or `{ senders: {}, last_updated: null }` if file missing/corrupt

    Export: `module.exports = { classifyBatch, classifyPipeline, isUnknownSender, updateSenderCache };`
