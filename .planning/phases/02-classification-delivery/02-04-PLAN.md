    **File 1: sandbox/skills/classify-email/index.js** (CommonJS module)

    This is the unified entry point that the comms agent heartbeat invokes.

    Imports:
    - `const { runSpamGateBatch } = require('./spam-gate');`
    - `const { classifyPipeline } = require('./classifier');`
    - `const { processClassifiedEmails, handleDigestReply: _handleDigestReply } = require('./delivery');`
    - `const { formatEmptyState, formatErrorState } = require('./digest-formatter');`

    **Function: `handleNewEmails(emails, account, telegramSendFn)`**
    - Input:
      - emails: array of `{ sender, subject, snippet, threadId, messageId }` (from Gmail API messages.get metadata)
      - account: 'personal' | 'work'
      - telegramSendFn: `async (text, parseMode) => telegramMessageId`
    - Pipeline flow per D-04 and RESEARCH Pattern 1:
      1. If `emails.length === 0`: call `telegramSendFn(formatEmptyState(0), 'HTML')` and return `{ total: 0 }`
      2. **Stage 1: Spam gate** — call `runSpamGateBatch(emails)` to get `{ spam, notSpam }`
      3. If all emails are spam: call `telegramSendFn(formatEmptyState(spam.length), 'HTML')` and return `{ total: emails.length, spam: spam.length, classified: 0 }`
      4. **Stage 2: Classification** — call `classifyPipeline(notSpam, account)` to get `{ classified, failed }`
      5. If classifyPipeline returns null (Gemini error):
         - call `telegramSendFn(formatErrorState('gemini_api', { count: notSpam.length }), 'HTML')`
         - return `{ total: emails.length, spam: spam.length, classified: 0, error: 'gemini_api' }`
      6. **Stage 3: Delivery** — call `processClassifiedEmails({ classified, failed }, telegramSendFn)`
      7. Return: `{ total: emails.length, spam: spam.length, classified: classified.length, failed: failed.length, delivery: deliveryResult }`
    - Error handling: wrap entire pipeline in try/catch. On unexpected error, send error notification via telegramSendFn and return error result.
    - Log each stage timing with `[classify-email]` prefix for debugging.

    **Function: `handleDigestReply(telegramMsgId, replyText)`**
    - Re-export from delivery.js: `return _handleDigestReply(telegramMsgId, replyText);`

    Export: `module.exports = { handleNewEmails, handleDigestReply };`

    **File 3: scripts/13-test-classification.sh**

    Bash script that:
    1. Loads test-emails.json
    2. For each email, runs it through the classification pipeline (skip Telegram delivery, just get classification)
    3. Compares predicted primary category to expected_categories (match = correct if predicted category is in expected list)
    4. Reports per-category precision: correct/total for each category
    5. Reports overall accuracy: total correct / 50
    6. Reports PASS if overall accuracy >= 80% (CLASS-07), FAIL otherwise
    7. Reports spam gate accuracy separately: how many expected spam_noise were caught by Ollama vs sent to Gemini

    The script should call a Node.js helper inline (here-doc or separate .cjs file) that:
    - Imports classifyBatch from classifier.js and runSpamGate from spam-gate.js
    - Processes each test email through both stages
    - Outputs JSON results to stdout for the bash script to parse

    Note: This script requires GEMINI_API_KEY in .env and Ollama running. It will make real API calls. With 50 emails in batches of 5, that is 10 Gemini requests (well within 250 RPD free tier). Add a note about this in the script header.

    Usage: `bash scripts/13-test-classification.sh` from project root.
